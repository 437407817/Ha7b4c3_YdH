





这是一个非常深入且关键的问题。我们需要区分两个完全不同的内存概念：**LVGL 绘图缓冲区（Draw Buffer）**和 **LTDC 显存帧缓冲区（Frame Buffer）**。

你的 `buf_1_1` 确实放在了单片机内部的 **AXI SRAM** ($0x24000000$) 中，但这只是“施工现场”，而不是“成品仓库”。

以下是为什么 LTDC 依然需要从 SDRAM 搬运数据的底层逻辑：

### 1. 缓冲区（Draw Buffer） vs 显存（Frame Buffer）

- **你的 `buf_1_1` (50行)：**

  它的作用是 LVGL 用来画图的临时空间。你设置了 50 行，大小是 $1024 \times 50 \times 4 \text{ 字节} = 200\text{ KB}$。它完全可以塞进 512KB 的 AXI SRAM。

- **LTDC 显存 (600行)：**

  屏幕要显示完整的画面，必须有一块内存存储全屏 **600 行**的数据。

  全屏大小是 $1024 \times 600 \times 4 \text{ 字节} = \mathbf{2.45\text{ MB}}$。

  **STM32H743 的内部 SRAM 没有任何一块连续区域能放下 2.45MB 的数据**（总共只有 1MB 左右，且分散在 AXI SRAM、SRAM1/2/3 中）。因此，这全屏的 2.45MB 必须放在外部 **SDRAM** 中。

### 2. 数据的“接力赛”过程

当你点击按钮或屏幕刷新时，数据是这样流动的：

1. **第一步（内部渲染）：** LVGL 在内部 AXI SRAM 的 `buf_1_1` 里画好这 50 行的按钮像素。
2. **第二步（数据搬运）：** LVGL 调用你的 `disp_flush` 函数，把这 50 行像素从内部 AXI SRAM 拷贝到外部 SDRAM 的对应位置。
3. **第三步（硬件显示）：** **这是最关键的一步。** LTDC 硬件控制器像一个勤劳的搬运工，每秒钟 60 次（60Hz 刷新率）不停地从 SDRAM 中读取那 2.45MB 的全屏数据，然后通过 RGB 接口发送给屏幕。

**结论：** 即使你的绘图缓冲区在内部，但最终 LTDC 必须去 SDRAM 搬运数据来维持屏幕显示。只要 LTDC 从 SDRAM 读数据迟到了哪怕几微秒，屏幕就会闪烁。

------

### 3. 为什么现在“不去按键也会自动闪”？

既然你不按键也在闪，说明 **LTDC 硬件在读取 SDRAM 时遭到了严重的“交通堵塞”**。

在 1024x600 ARGB8888 模式下，SDRAM 的总线几乎是满负荷的。如果出现自动闪烁，通常是以下两个原因：

#### 原因 A：LTDC 抢不到总线（优先级太低）

H7 的 AXI 总线矩阵中，多个外设（CPU、DMA2D、LTDC、以太网等）都在抢 SDRAM 的访问权。默认情况下，LTDC 的优先级不够高。

**解决方法：** 提升 LTDC 的 AXI 优先级（见下文代码）。

#### 原因 B：像素时钟（Pixel Clock）太快

如果像素时钟频率设为 50MHz 或更高，SDRAM 可能来不及供应数据。

**解决方法：** 在 `bsp_lcd.c` 中，尝试将像素时钟降低到 **33MHz 或 40MHz**。虽然刷新率会变成 45Hz 左右，但通常能彻底消除闪烁。

------

### 4. 解决“自动闪”的代码补丁

请尝试在 `main.c` 初始化完 LCD 后，加入这段“强行提速”的代码：

C

```
/* 提升 LTDC 在总线矩阵中的优先级，防止被 CPU 抢占导致闪屏 */
void LTDC_Priority_Config(void)
{
    /* 开启 D2 域 SRAM1 时钟（AXI 优先级寄存器在此域） */
    __HAL_RCC_D2SRAM1_CLK_ENABLE();

    /* 设置 LTDC 为 AXI 总线最高优先级 */
    /* 寄存器地址 0x51003000 是总线优先级配置起始地址 */
    *((uint32_t *)(0x51003000 + 0x000)) = 0x00000001; // 设置 LTDC 访问 AXI 总线的优先级
}
```

### 总结建议：

1. **确认全屏显存地址：** 检查你的 LTDC 初始化代码（通常在 `bsp_lcd.c`），看 `LayerCfg.FBStartAdress` 是否指向了 `0xD0000000` (SDRAM)。
2. **降低像素时钟：** 降低时钟是解决大分辨率、高色深闪屏最有效、最简单的物理方案。
3. **优先级提权：** 必须让 LTDC 成为总线上的“霸王”，否则 CPU 只要一工作，LTDC 读 SDRAM 就会被打断。

下面给你一套 **STM32H743 + D-Cache + DMA 必用的标准缓存操作代码**，直接复制就能用，适用于 **HAL 库 / 裸机**。

# 一、核心缓存操作函数（Cortex-M7 通用）
```c
#include "core_cm7.h"

/**
 * @brief  清空指定地址缓存（Cache -> 内存，写回）
 * @param  addr: 数据起始地址（尽量32字节对齐）
 * @param  len: 长度
 */
void SCB_CleanDCache_by_Addr(uint32_t addr, uint32_t len)
{
    uint32_t end = addr + len;
    addr &= ~(SCB_DCACHE_LINE_SIZE - 1);  // 32字节对齐

    for (; addr < end; addr += SCB_DCACHE_LINE_SIZE)
    {
        SCB->DCCMVAC = addr;
    }
    __DSB();
}

/**
 * @brief  失效指定地址缓存（内存 -> Cache，丢弃旧缓存）
 * @param  addr: 数据起始地址
 * @param  len: 长度
 */
void SCB_InvalidDCache_by_Addr(uint32_t addr, uint32_t len)
{
    uint32_t end = addr + len;
    addr &= ~(SCB_DCACHE_LINE_SIZE - 1);

    for (; addr < end; addr += SCB_DCACHE_LINE_SIZE)
    {
        SCB->DCIMVAC = addr;
    }
    __DSB();
}
```

# 二、DMA 使用场景（最关键，90% 问题都在这）
## 场景1：CPU 写数据 → DMA 发送（如 SPI/UART/ADC/DAC 发送）
**必须先 Clean**，让缓存写回内存，DMA 才能读到最新数据。
```c
// 1. CPU 填充数据
uint8_t tx_buf[1024];
memset(tx_buf, 0xAA, 1024);

// 2. 清缓存：Cache → RAM
SCB_CleanDCache_by_Addr((uint32_t)tx_buf, sizeof(tx_buf));

// 3. 启动DMA发送
HAL_UART_Transmit_DMA(&huart1, tx_buf, sizeof(tx_buf));
```

## 场景2：DMA 接收数据 → CPU 读取
**必须先 Invalidate**，丢弃旧缓存，让 CPU 重新从内存读。
```c
// 1. 启动DMA接收
uint8_t rx_buf[1024];
HAL_UART_Receive_DMA(&huart1, rx_buf, sizeof(rx_buf));

// 2. 等待DMA完成
// ...

// 3. 失效缓存：强制CPU从内存读取最新数据
SCB_InvalidDCache_by_Addr((uint32_t)rx_buf, sizeof(rx_buf));

// 4. CPU 处理 rx_buf
```

## 场景3：双缓冲 / 多次 DMA 收发
完整流程：
```c
// 发送
SCB_CleanDCache_by_Addr(...);
HAL_XXX_DMA(...);
while(传输忙);

// 接收
while(传输忙);
SCB_InvalidDCache_by_Addr(...);
// 处理数据
```

# 三、最容易踩的坑
1. **缓存行是 32 字节**
   地址不对齐会**误清相邻变量**，数组最好：
   ```c
   uint8_t buf[2048] __attribute__((aligned(32)));
   ```

2. **外设地址不要进 Cache**
   如 `0x40000000` 以上外设寄存器，**不要用 Cache 操作**，否则死机/数据错乱。

3. **MDMA、BDMA 同样要处理**
   只要是**不透过CPU访问内存**的总线主控，都要清缓存。

# 四、最简单的万能模板
你以后直接按这个套：
```c
// DMA 发送前
SCB_CleanDCache_by_Addr((uint32_t)buf, len);

// DMA 接收完成后
SCB_InvalidDCache_by_Addr((uint32_t)buf, len);
```
